# 🚀 Deployment Notes

Herb AI uses a **three-tier cloud deployment** architecture across three separate free-tier platforms.

---

## Architecture Overview

```
User Browser
     │
     ▼
┌─────────────────────┐
│  Frontend (Vercel)  │  ← React + Vite
│  myherb-live.vercel │
└────────┬────────────┘
         │ REST API calls
         ▼
┌─────────────────────┐
│  Backend (Render)   │  ← Node.js / Express
│  MongoDB Atlas      │  ← Cloud Database
└────────┬────────────┘
         │ @gradio/client
         ▼
┌─────────────────────────────────┐
│  ML Model (Hugging Face Spaces) │  ← PyTorch Fusion CNN + Gradio
│  Ronaldo-7/MlendHerb            │
└─────────────────────────────────┘
```

---

## 1. 🤖 ML Model — Hugging Face Spaces

**URL:** https://huggingface.co/spaces/Ronaldo-7/MlendHerb

The ML model (`model.pt`) is a TorchScript Fusion CNN (ResNet50 + InceptionV3) and is too large to host on Render's free tier due to RAM constraints. It is deployed as a Gradio app on Hugging Face Spaces.

### Files required in the Space:
```
app.py
model.pt
requirements.txt
```

### `requirements.txt`:
```
torch
torchvision
Pillow
gradio
```

### How it's called from the Frontend:
```js
import { Client } from "@gradio/client";

const client = await Client.connect("Ronaldo-7/MlendHerb");
const result = await client.predict("/predict", { image: selectedImage });
const prediction = result.data[0]; // { class: "...", probability: 0.98 }
```

> ⚠️ **Cold Start Warning:** Free-tier HF Spaces go to sleep after inactivity. The first request after idle may take 30–60 seconds to respond.

---

## 2. ⚙️ Backend — Render

**URL:** Set as `VITE_BACKEND_URL` in Vercel environment variables.

The Node.js/Express backend handles authentication, herb data, queries, chatbot, and email. It connects to MongoDB Atlas.

### Environment Variables (set in Render dashboard):
```env
MONGO_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/myherb
JWT_SECRET=your_jwt_secret_key
EMAIL_USER=your_gmail@gmail.com
EMAIL_PASS=your_gmail_app_password
GEMINI_API_KEY=your_google_gemini_api_key
PORT=5000
```

### Notes:
- Uses **MongoDB Atlas** (free M0 cluster) as the database — not a local MongoDB instance.
- Gmail SMTP requires an **App Password** (not your regular Gmail password). Enable 2FA on your Google account, then generate an App Password at myaccount.google.com/apppasswords.
- Render free tier **spins down after 15 minutes of inactivity** — first request after idle may be slow (~30s).

---

## 3. 🌐 Frontend — Vercel

**URL:** https://myherb-live.vercel.app

The React + Vite frontend is deployed on Vercel with automatic deployments triggered on every push to the `master` branch.

### Environment Variables (set in Vercel dashboard):
```env
VITE_BACKEND_URL=https://your-render-backend-url.onrender.com
```

### Notes:
- The `@gradio/client` package must be listed in `package.json` dependencies (not devDependencies) for Vercel to install it during build.
- Vite cache issues after dependency changes: delete `node_modules/.vite` and restart dev server.

---

## 4. 🗄️ Database — MongoDB Atlas

The backend connects to a free **M0 MongoDB Atlas cluster**.

### Setup steps:
1. Create a free cluster at [mongodb.com/atlas](https://mongodb.com/atlas)
2. Add your Render server's IP (or `0.0.0.0/0` for all IPs) to the **IP Access List**
3. Create a database user and copy the connection string to `MONGO_URI` in Render

## Local Development

For running the project locally, refer to the original development repo:
👉 https://github.com/Tawab44/myherb

```bash
# Backend
cd Backend
npm install
node server.js   # runs on http://localhost:5000

# Frontend
cd Frontend
npm install
npm run dev      # runs on http://localhost:5173

# ML Model (local)
cd MLend
pip install -r requirements.txt
uvicorn app:app --reload   # runs on http://localhost:8000
```

> For local ML, update `Home.jsx` to use `http://127.0.0.1:8000/predict/` with `axios` instead of the Gradio client.

## Free Tier Limitations Summary

| Platform | Limitation | Impact |
|----------|-----------|--------|
| Hugging Face Spaces | Sleeps after inactivity | ~30–60s cold start on first prediction |
| Render | Spins down after 15 min idle | ~30s delay on first API call |
| MongoDB Atlas M0 | 512MB storage, shared CPU | Fine for development/demo scale |
| Vercel | 100GB bandwidth/month | No practical limit for this project |
